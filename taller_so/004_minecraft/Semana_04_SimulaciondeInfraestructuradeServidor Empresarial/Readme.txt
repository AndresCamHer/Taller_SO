Simulación en Minecraft – Semana 4
Infraestructura de Servidor Empresarial

Este proyecto representa la infraestructura de una empresa y sus servicios de red, construida en Minecraft como evidencia práctica. El diseño se realizó en un edificio pequeño, pero suficiente para incluir las tres áreas solicitadas: Administración, Ventas y Sistemas. Aunque la escala es reducida y no se representaron físicamente los 50 empleados, cada sala y servicio simboliza cómo se organizarían los recursos y el control de acceso para una empresa de ese tamaño.

La sala de Administración representa la gestión de documentos y archivos compartidos. Se colocaron escritorios, sillas y un cofre con libros y mapas como símbolo de la información administrativa. Libreros y cuadros complementan la idea de organización.

La sala de Ventas representa el área de reuniones y exposición de productos. Se construyó una mesa grande con sillas alrededor y un podio para presentaciones. Un cofre con ítems variados simboliza los productos, mientras que mapas y cuadros en la pared simulan gráficas de ventas.

La sala de Sistemas representa el centro de control tecnológico. Se colocaron mesas con “computadoras” usando cuadros, botones y placas. Redstone, palancas y botones simulan paneles de control. Un cofre protegido simboliza los recursos técnicos con acceso restringido, y mapas en la pared representan el monitoreo de la red.

La sala de servidores (Data Center) es un espacio cerrado con bloques de comando y bloques de luz, simbolizando racks y energía. El acceso está restringido, identificado con un cartel en la entrada. Los bloques de comando representan los servidores principales y los bloques de luz la energía del centro de datos. Este espacio refleja servicios como Active Directory, DNS, DHCP, File Server y Hyper-V.

Los servicios de Windows Server se representaron de la siguiente manera:
- Active Directory: sala de control central con libro de usuarios.
- DNS: antena en la azotea con cartel DNS.
- DHCP: cofres con recursos distribuidos.
- File Server: cofres exclusivos por departamento.
- Hyper-V: área adicional con bloques que representan servidores virtuales.

El control de acceso se simuló mediante puertas con mecanismos de redstone y cofres protegidos. Solo el área de Sistemas tiene acceso a ciertos recursos, lo que refleja cómo en Windows Server se asignan permisos y roles.

Se construyeron dos servidores: uno pequeño (mínimo) y uno grande (recomendado). El servidor pequeño representa los requisitos básicos de instalación, mientras que el grande simboliza los requisitos óptimos para soportar la empresa. El servidor grande puede manejar mejor la carga de usuarios, mientras que el pequeño podría colapsar si se destruye parte de su estructura, simulando una caída del sistema.

Conclusión: Aunque el edificio es pequeño y la representación es simbólica, cada sala cumple con la función académica requerida. Al realizar esta práctica se comprendió cómo los servicios de Windows Server se relacionan con la organización de una empresa. Minecraft permitió visualizar de manera sencilla conceptos como control de usuarios, carpetas compartidas, respaldo de información y seguridad de acceso. La actividad refuerza la comprensión visual de la infraestructura y la relación entre teoría y práctica.
